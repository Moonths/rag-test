## 项目说明文档（RAG 工业化流水线视角）

你可以把这个项目理解成一条“**文档 → 知识库 → 检索 → 生成 → 追溯**”的流水线：  
离线阶段把文档加工成可检索的知识；在线阶段把“用户问题”送入检索与生成，并把证据（引文）一起回传。

> **阅读建议（初学者路径）**  
> `backend/app/app_main.py` → `backend/app/router/chat_rt.py` → `backend/app/service/core/file_parse.py` → `backend/app/service/core/retrieval.py` → `backend/app/service/core/chat.py` → `frontend/src/pages/chat/index.tsx`

---

## 1. 项目目标与能力边界

- **项目目标**：构建一个“智能文档问答系统”，支持用户上传文档后基于知识库进行问答，并在回答中携带引用来源，降低幻觉、增强可解释性。
- **核心能力**
  - **永久知识库**：上传文档 → 解析 → 分块 → 向量化 → 写入 Elasticsearch（支持语义检索+关键词检索）。
  - **临时文档**：小文档快速解析 → 存入 Redis（2 小时过期），可在当次会话中直接被问答使用（不做向量化）。
  - **在线对话**：混合检索召回证据 → Prompt 组装 → 大模型流式输出（SSE）→ 落库（会话/消息/引文/推荐问题/思考过程）。

---

## 2. 系统架构总览（组件分工）

### 2.1 运行时组件

- **前端（Vite + React + TS）**
  - 主要页面：`frontend/src/pages/chat/index.tsx`
  - 通过 `Authorization: Bearer <token>` 调用后端，并以 **SSE 流**形式渲染回答（打字机效果）。
- **后端（FastAPI）**
  - 总入口：`backend/app/app_main.py` 注册路由。
  - 核心路由：`backend/app/router/chat_rt.py`
  - 用户与鉴权：`backend/app/router/user_rt.py` + `backend/app/service/auth.py`
  - 历史与文件：`backend/app/router/history_rt.py`
- **PostgreSQL**
  - 保存用户、会话、消息、知识库元信息、上传记录等结构化数据。
- **Elasticsearch（8.x）**
  - 保存分块后的文本 + 向量 + 元信息，用于混合检索与重排序。
  - 该项目使用“**每个用户一个索引**”的思路：检索/写入时索引名常用 `user_id`（见 `retrieve_content(indexNames=user_id, ...)`）。
- **Redis**
  - 保存“快速解析”的临时文档内容（key 常用 `session_id`，2 小时过期）。

### 2.2 Docker Compose 服务

见 `backend/docker-compose.yml`：

- **swxy_api**：FastAPI 主服务，端口 `8000`
- **gsk_pg**：PostgreSQL，端口 `5432`
- **es01**：Elasticsearch
- **redis**：Redis

## 3. 端到端流水线（你需要掌握的主线）

我把整体拆成两条流水线：**离线（建库）**与**在线（问答）**。你只要抓住“每一步输入/输出/存储点”，就能快速读懂项目。

### 3.1 离线流水线：上传文档 → 解析/分块/向量化 → 入库（ES + PG）

**触发接口**：`POST /upload_files`（`backend/app/router/chat_rt.py`）

**步骤**

- **S1 接收文件并落盘**
  - 保存路径：`storage/file/{session_id}/{file_name}`
- **S2 文档解析 + 分块（Chunking）**
  - 入口：`execute_insert_process(file_url, file_name, session_id)`  
  - 文件：`backend/app/service/core/file_parse.py`
  - 内部调用 `service/core/rag/app/naive.py` 的 `chunk()`（解析 PDF/DOCX/TXT/Excel 并生成 chunk）
- **S3 向量化（Embedding）**
  - `batch_generate_embeddings()` → `generate_embedding(texts)`
  - 将每个 chunk 文本转成向量（默认 DashScope embedding，见环境变量）
- **S4 写入 Elasticsearch**
  - `ESConnection.insert(documents=..., indexName=index_name)`
  - 存储内容：chunk 文本、分词字段、doc_id/docnm、向量字段 `q_{dim}_vec` 等
- **S5 写入 PostgreSQL 元信息**
  - `insert_knowledgebase(user_id, file_name)`
  - 用于“知识库文件列表/删除”等管理能力

**这一条流水线的产物**

- ES：可检索的分块知识（供在线检索使用）
- PG：文件元信息（列表/去重/删除）
- 本地磁盘：原始文件（可用于回溯与重建）

---

### 3.2 在线流水线：用户提问 → 检索证据 → Prompt 组装 → 大模型流式生成（SSE）→ 落库

**触发接口**：`POST /chat_on_docs?session_id=...`（`backend/app/router/chat_rt.py`）

**步骤**

- **S1 鉴权与会话定位**
  - 前端携带 `Authorization: Bearer <token>`
  - 后端用 `access_security` 从 token 中取出 `user_id`
- **S2 检索（Retrieval）**
  - 入口：`retrieve_content(user_id, question)`
  - 文件：`backend/app/service/core/retrieval.py`
  - 特征：混合检索（关键词 + 向量），并做 rerank，返回 Top-K chunks
- **S3 证据增强（Augment）**
  - 从 Redis 取“快速解析文档内容”：`get_quick_parse_content(session_id)`
  - 将“ES 检索 chunks + Redis 临时文档”统一整理为 references 列表
- **S4 Prompt 组装**
  - 文件：`backend/app/service/core/chat.py`
  - 策略：要求模型基于参考资料回答，并在内容中标注引用（形如 `##引用编号$$`）
- **S5 大模型流式生成（Generation）**
  - `OpenAI(... base_url=DASHSCOPE_BASE_URL)` 走 OpenAI 兼容接口
  - 主回答模型：`deepseek-r1`，使用 `stream=True`
  - 返回给前端：SSE 事件流 `text/event-stream`
    - 先推：`{"documents": [...]}`（让前端先渲染“引用来源/文档列表”）
    - 再逐 token 推：`{"content": "...", "thinking": true/false}`
    - 最后推：`event: end data: [DONE]`
- **S6 产出“推荐问题”**
  - `generate_recommended_questions()` 使用 `qwen2.5-7b-instruct`
  - 推送：`{"recommended_questions": [...]}`（同样走 SSE）
- **S7 落库**
  - `write_chat_to_db()`：写入 `messages`（包含 documents/recommended_questions/think）
  - `update_session_name()`：新会话第一次提问时生成会话名（`qwen2.5-72b-instruct`）

---

### 3.3 临时文档流水线：快速解析 → Redis（2小时）

**触发接口**：`POST /quick_parse?session_id=...`

- 读取上传文件二进制
- 调用 `quick_parse_service.quick_parse_document(...)`
- 解析结果写入 Redis（key 通常是 `session_id`）
- 同时在 PG 中记录上传记录（用于会话内“文档列表/摘要”能力）

**读取临时文档**：`GET /get_parsed_content?session_id=...`

## 4. 前端如何对接（你看到的“打字机效果”从哪来）

### 4.1 核心接口封装

文件：`frontend/src/api/session.ts`

- `session.create()` → `POST /create_session`
- `session.chat({id,message})` → `POST /chat_on_docs?session_id=id`  
  - 请求头：`Accept: text/event-stream`
  - `responseType: 'stream'` + `adapter: 'fetch'`

### 4.2 流式解析逻辑

文件：`frontend/src/pages/chat/index.tsx`

- 通过 `ReadableStream.getReader()` 循环读
- 以换行 `\n` 分割出 SSE 行
- 只处理以 `data: ` 开头的行
- JSON 内根据字段做分流：
  - `thinking: true` → 写入 `think`
  - 否则 → 写入 `content`
  - `documents` / `recommended_questions` → 更新右侧栏与推荐区

### 4.3 统一鉴权头

文件：`frontend/src/api/request/plugins/auth.ts`

- 自动把 `userState.token` 写入请求头：`Authorization: Bearer <token>`
- 401 自动清 token 并跳转登录

## 5. 后端 API 清单（按业务分组）

### 5.1 用户与认证（`backend/app/router/user_rt.py`）

- `POST /register`：注册
- `POST /login`：登录（返回 JWT）
- `POST /sts-token`：语音服务临时凭证（可忽略不影响 RAG 主流程）

### 5.2 对话与文档（`backend/app/router/chat_rt.py`）

- `POST /create_session`：创建会话
- `POST /quick_parse?session_id=`：临时文档快速解析（Redis）
- `GET /get_parsed_content?session_id=`：读取临时解析内容
- `POST /upload_files?session_id=`：上传到永久知识库（ES + PG）
- `POST /chat_on_docs?session_id=`：RAG 问答（SSE 流式输出）
- `GET /sessions/{session_id}/documents`：会话文档上传记录列表
- `GET /sessions/{session_id}/documents/summary`：会话文档摘要

### 5.3 历史与知识库管理（`backend/app/router/history_rt.py`）

- `GET /get_sessions`：会话列表
- `GET /get_messages?session_id=`：会话消息列表
- `GET /get_files`：知识库文件列表
- `DELETE /delete_file/{file_name}`：删除知识库文件（ES + 本地文件 + PG）

## 6. 配置与环境变量（最容易踩坑的地方）

### 6.1 `.env`/`.env.example`

后端示例见 `backend/.env.example`，你至少需要配置：

- **DASHSCOPE_API_KEY**：阿里云 DashScope key（用于 deepseek-r1、qwen、embedding）
- **DASHSCOPE_BASE_URL**：默认 `https://dashscope.aliyuncs.com/compatible-mode/v1`
- **DATABASE_URL**：PostgreSQL 连接串
- **ES_HOST**：ES 地址（docker 内一般是 `http://es01:9200`）
- **ELASTIC_PASSWORD**：ES 密码（compose 里会启用安全）

### 6.2 Docker Compose 里的关键点

文件：`backend/docker-compose.yml`

- **nltk_data 挂载**：需要把本机的 `nltk_data` 路径改成你的（否则分词/解析可能报错）
  - 示例：`/Users/maojike/nltk_data:/usr/local/nltk_data`

## 8. 常见问题（面向初学者的排错清单）

- **前端调用 401**
  - 检查是否已登录获取 token
  - 检查请求头是否带 `Authorization: Bearer <token>`（前端 auth 插件负责自动注入）
- **ES 连接失败**
  - docker 内访问 ES 通常是 `http://es01:9200`，不要用 `localhost`
  - 检查 `ELASTIC_PASSWORD` 与 ES 安全配置
- **上传解析失败 / 分词报错**
  - 优先检查 `nltk_data` volume 是否正确挂载
- **流式回答不显示**
  - 后端 `media_type="text/event-stream"`
  - 前端 `adapter: 'fetch'` + `responseType: 'stream'` 是否生效
  - 浏览器控制台是否有跨域/代理问题（CORS）
